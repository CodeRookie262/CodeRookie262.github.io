# 前方核能 还可以这样吗

在开始之前,给大家一道比较经典却依然坑了很多人的题.

    for(var i = 0;i < 5;i++){
        setTimeout(function(){
            console.log(i);
        },i*1000);
    }
    console.log(i);

会依次输出什么呢?请选择一个结果:

  • [ ] A. 0, 1, 2, 3, 4, 5
  • [ ] B. 5, 0, 1, 2, 3, 4
  • [ ] C. 5, 5, 5, 5, 5, 5
    请认真动脑思考后再看下面部分,严肃脸!

聪明的你会选择哪个选项呢?
选 A 的同学心里应该是这样想的, js 代码执行顺序至上而下,所以应该要等 for 循环全部执行完毕在执行 for 循环后面的代码,所以也就是 0, 1, 2, 3, 4, 5

选 B 的同学应该是这样想的,虽然 A 选项中的 for 循环先执行是这样没错,但是 for 循环里面的 setTimeout 属于异步代码,会被堆在同步代码后面,所以应该是 5, 0, 1, 2, 3, 4

最后排除 A 和 B ,选 C 的同学,应该是这样想的, A 选项没有区分异步与同步的执行区别, B 选项,虽然正确的认识到了 异步与同步代码的区别,但是却没有意识到作用域的问题,虽然 for 循环是同步的,定时器属于异步代码,所有当你执行定时器的时候,此时的 for 循环已经执行完毕了,所以 i 的值也是最后循环结束的那个 i,也就是 5
那么,你是终于那一种的呢,上面的几种虽然都有道理,因为笔者是过来人,以上的想法都有过,不过正确答案只有一个,就是 C

那么,很多同学想必都遇到过这个坑,就是代码是在 for 循环里面写代码,例如定时器或者点击等事件,当触发事件是,结果总于预期差入很大,就是取到的数据是 for 循环结束后的数据,这个方法该怎么解决呢,下面就针对定时器来讲一下如何解决这个坑,如何让定时器一次输出 0, 1, 2, 3, 4, 5

# bind()

    for(var i = 0;i < 5;i++){
        setTimeout(function(i){
            console.log(i)
        }.bind(null,i),i*1000);
    }
    setTimeout(function(){
        console.log(i)
    },i*1000);
    //结果 0,1,2,3,4,5

# 闭包

利用IIFE来解决 i 的问题

    for(var i = 0;i < 5;i++){
        (function(j){
            setTimeout(function(){
                console.log(j);
            },j * 1000);
        })(i)
    }
    setTimeout(function(){
        console.log(i)
    },i*1000);
    //结果 0,1,2,3,4,5

# let

利用ES6 的(Block Scope)块级作用域let来处理

    for(let i = 0;i < 5;i++){
        setTimeout(function(){
            console.log(i);
        },i * 1000);
    }
    setTimeout(function (){
        console.log(i)
    },i * 1000);
    //结果: 先抛出一个错 ReferenceError: i is not defined 然后输出 0,1,2,3,4

# Promise

Promise 是处理异步代码的优秀方案
第一种方式

    function fn(num){
		if(num > 5) return false;
		new Promise(function(resolve){
			console.log(num);
			setTimeout(resolve,1000,++num);
		})
		.then(function(res){
			fn(res)
		}) 
	}
	fn(0);
	//结果 0,1,2,3,4,5

# async

如果说Promise是解决异步的最佳解,async就是promise的超级加强版,async的强大之处在于它的await,直接 Show Code

    (async function(){
        for(var i = 0;i < 5;i++){
            await new Promise(function(resolve){
                setTimeout(resolve,1000);
            });
            console.log(i);
        }
        
        await new Promise(function(resolve){
            setTimeout(resolve,1000);
        });
        console.log(i)
    })()
