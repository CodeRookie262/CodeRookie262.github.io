    # 任务队列

    首先,我们需要明白以下几件事:

    • js分为 异步任务 和 同步任务
    • 同步任务 在主线程上执行,形成一个执行栈
    • 主线程之外,事件触发线程管理着一个任务队列,只要一部任务有了运行结果,就在任务队列中放置一个事件
    • 一旦执行栈中的所有 同步任务 执行完毕,系统就会读取任务队列,将可运行的异步任务添加到执行栈中,开始执行

    事件循环是根据任务队列的机制进行协调的. 一个 Event Loop中,可以有多个任务队列(task queue),一个任务队列便是一系列有序任务的集合;每个人物都有哟个任务源(task source),源于同一个任务源的task必须放到同一个任务队列,所以根据任务源的不同,其对应的task放置的队列也就随着不同

    想了解更多关于 Event Loop(事件循环)的可以前往公众号往期文章《搞懂Event Loop》

    # 宏任务

    Marco Task(宏任务),可以理解为事件队列中的每一个事件都是一个macrotask

    Marco Task 主要包括: script(整体代码), setTimeout,setInterval, I/O, UI交互事件, postMessage 和 setImmediate(NodeJS) 等

    # 微任务

    Micro Task(微任务),可以理解为当前 task 执行结束后立即执行的任务,也就是在当前任务后,下一个task前

    所以他的响应速度相比setTimeout,因为无需等待渲染,也就是说,在某一个 macrotask 执行结束后,就会将他执行期间所产生的所有 microtask 都执行掉

    microtask 主要包括 : promise.then 和 process.nextTick(NodeJS)

    # 运行机制

    [ 采摘至 简书 ]

    在事件循环中,每进行一次循环操作称为 tick,每一次 tick 的任务处理模型是比较复杂的,但步骤如下:

    • 执行一个宏任务(栈中没有就从任务队列中获取)
    • 执行过程中如果遇到微任务,就将它添加到微任务的任务队列中
    • 宏任务执行完毕后,立刻执行当前微任务的所有微任务
    • 当前宏任务执行完毕,开始检查渲染,然后 GUI 线程接管渲染
    • 渲染完毕后,JS线程继续接管,开始下一个宏任务(从事件队列获取)

    # Promise 和 async中的立刻执行

    我们知道 Promise 中的异步体现在 then 和 catch 中,所以写在 Promise 中的代码是被当成同步任务立刻执行的. 而在 async/await 中,在 await出现之前的代码也是被当成同步任务立刻处理的. 那么出现 await 时发生了什么呢?

    await 做了什么

    顾名思义,await 可以理解为 async wait 的简写, 有等待的意思,await 等待的是一个表达式,这个表达式的返回值可以是一个 Promise 对象也可以是其他值

    很多人以为 await 会一直等待之后的表达式执行完毕之后才会执行后面的代码. 但是经过测试和查阅许多资料后才知道,其实实际上 await 是一个让出线程的标志. await后面的表达式会先执行一遍,将await后面的代码加入到 microtask ,然后跳出整个 async 函数且执行async 函数后面的代码

    了解了这么多,可以回想下文章头的那道题,一步一步解析一下为何是你个结果

    console.log(`async1 start`)
    await async2()
    console.log(`async1 end`)

async1()

    1. 首先,事件循环从宏任务(marcotask)队列开始,整个时候宏任务队列中只有一个 script (整体代码);当遇到任务源(task source),则会先方法任务到对应的任务队列中去

    2. 然后我们看到首先定义了两个async函数,之后遇到了 console 语句输出 script start,输出语句之后,script继续往下执行,读取到setTimeout,它是宏任务的任务源,则将其任务派发到对应的任务队列中去

    3. script继续往下执行,执行了async1函数,函数内部的await之前的代码会被立刻执行,所以会立刻输出async1 start.同时,await后面的表达式会被执行一遍,所以会输出async2,然后将await后面的代码加入到 microtask 的队列中去,紧接着跳出async1函数来执行后面的代码

    4. script任务继续执行,遇到 Peomise 对象的实例.因为 Promise 中的函数是立刻执行,所以也会立刻输出 promise1,然后执行resolve更改为成功状态,将 Promise.then 中的函数分配到 microtask 任务中

    5. script任务继续往下执行,最后输出script end,然后第一次的宏任务执行完毕,开始在下一次宏任务执行之前执行清空微任务中的事件队列 因此在 script任务执行完毕之后,开始查找清空微任务队列,此时微任务中的队列有 async1 end和promise2两个任务,因此按顺序输出async1 end和promise2,当microtask 执行完毕就表示第一轮循环结束了,开始第二轮循环

    6. 第二轮循环依旧是从宏任务队列开始的,此时宏任务只有一个setTimeout任务,取出来直接输出,最后流程结束
