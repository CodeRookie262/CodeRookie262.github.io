# 车门已焊死,带你了解观察者模式和订阅发布模式

观察者模式

当对象间存在一对多关系时，则使用观察者模式（Observer Pattern）。比如，当一个对象被修改时，则会自动通知它的依赖对象。观察者模式属于行为型模式。新伟性模式关注的是对象间的通讯,观察者模式就是观察者与被观察者之间的通讯。

观察者模式还有一个别名叫做“发布订阅模式”，吧订阅者和订阅目标联系在一起，当订阅目标发生改变的时候，就会逐个通知订阅者。有点类似一个软件更新版本了，会通知用户版本更新信息，告知用户及时更新，软件的更新通知和用户就是一对多的依赖关系

发布订阅模式

有看过或了解过设计模式的人可能会发现里面没有发布订阅模式这种模式，其实，他归属与观察者模式，由于它得到了很多开发者的青眯，已经独立于观察者模式，成为另外一种设计模式。

现在的发布订阅模式中，成为发布者的消息发送不会直接将消息发送给订阅者，在发布者和订阅者之间存在着第三个组件，成为消息代理或调度中心或中间件，它维持着发布者和订阅者之间的联系，过滤所有发布者所传入的消息并相应地把它们分发给订阅者。

类似你在某站关注了某位UP主，当该UP主发布动态的时候，某站就会为你们推送做一条动态。其中该UP主就是发布者，你就是订阅者，而某站就是调度中心，你和关注的UP主没有直接的消息往来的，都是通过某站来协调的

观察者模式

 class FairyTaill{
        constructor(name,level){
          this.name = name
          this.level = level
          this.partners = []
        }
        subscript(member,fn){
          console.log(`${this.level}级成员${this.name}加入了${member.name}的阵营`)
          member.partners.push(fn)
        }
        publish(task,money){
          console.log(`${this.level}级成员${this.name}发布了${task}的任务,任务酬劳:${money || 0}J`)
          console.log("=========================="+ task +"=================================");
          this.partners.forEach(item => item())
          console.log("=====================================================================");
        }
        unsubscript(member,fnName){
          let fnIndex = member.partners.findIndex(item => item.name.includes(fnName))
              if(fnIndex == -1){
                return false;
              }
              member.partners.splice(fnIndex,1)
              console.log(`${this.name}退出了${member.name}的任务队列`)
        }
      }
      const President = new FairyTaill("马克洛夫","SSS")

      const Lucy = new FairyTaill("露西","A")
            Lucy.subscript(President,function(){
              console.log(`${this.name}: "终于又可以组队了~"`)
            }.bind(Lucy))

      const Naz = new FairyTaill("夏","S")
            Naz.subscript(President,function(){
              console.log(this.name + ":'我燃烧起来了!'")
            }.bind(Naz))

      const Habi = new FairyTaill("哈比","C")
            Habi.subscript(President,function habi(){
              console.log(this.name + ":'爱!'")
            }.bind(Habi))
            
            Habi. unsubscript(President,"habi")
            President.publish("大魔斗争霸赛","3000w")

发布订阅模式

const FiaryTaill = {
      type: "competition",
      topics: {},
      subscript(topic,fn){
        if(!this.topics[topic]){
          this.topics[topic] = []
        }
        this.topics[topic].push(fn)
      },
      publish(topic,money){
        if(!this.topics[topic]){
          console.log(`没有人接收任务${topic}`)
          return false;
        }
        this.topics[topic].forEach(item => item(money))
      },
      unsubscribe(topic,fnName){
          
             
              let fnIndex = this.topics[topic].findIndex(item => item.name.includes(fnName))
              if(fnIndex == -1){
                return false;
              }

              this.topics[topic].splice(fnIndex,1)

      }
    }   

    class Member{
      constructor(name,level){
        this.name = name
        this.level = level
      }
      subscript(topic,fn){
        console.log(`${this.level}级魔导师 ${this.name} 订阅了 ${topic} 任务`)
        FiaryTaill.subscript(topic,fn)
      }
      publish(topic,money){
        console.log(`${this.level}级魔导师 ${this.name} 发布了 ${topic} 任务`)
        FiaryTaill.publish(topic,money)
      }
      unsubscribe(topic,fn){
        console.log(`${this.name}取消了${topic}任务`)
        FiaryTaill.unsubscribe(topic,fn)
      }
    }

    const President = new Member("马克洛夫","SSS")
    const Lucy = new Member("露西","A")

          Lucy.subscript("大魔斗",function LucyFn(){
            console.log(`${this.name}: "终于又可以组队了~"`)
          }.bind(Lucy))

          Lucy.unsubscribe("大魔斗","LucyFn")


    const Naz = new Member("夏","S")
          Naz.subscript("大魔斗",function(){
            console.log(this.name + ":'我燃烧起来了!'")
          }.bind(Naz))

    const Habi = new Member("哈比","C")
          Habi.subscript("大魔斗",function(){
            console.log(this.name + ":'爱!'")
          }.bind(Habi))

          President.publish("大魔斗","3000w")

观察者模式与发布订阅模式对打的区别就是发布订阅模式有个事件调度中心

观察者模式由具体目标调度,每个订阅的目标都需要处理观察者,这种处理方式比较简单粗暴,造成代码冗余;

而发布订阅模式中统一由调度中心进行处理,订阅者和发布者互不干扰,消除了发布者与订阅者之间的相互依赖,一方面实现了解耦,同时是可以实现更细粒度的一些控制,比如发布者发布了很多类型的动态,但是不太喜欢所有订阅者都可以收到他的动态,这时就可以在调度执行做一些处理,类似权限控制之类的,和微信朋友圈差不多。同时还可以做一些节流操作。
