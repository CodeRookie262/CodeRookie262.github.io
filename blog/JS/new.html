# 利用"交叉观察者"轻松实现触底加载, 懒加载以及吸顶盒

在阅读之前来一波MDN解说:   IntersectionObserver接口 (从属于Intersection Observer API) 提供了一种异步观察目标元素与其祖先元素或顶级文档视窗(viewport)交叉状态的方法。祖先元素与视窗(viewport)被称为根(root)。

  当一个IntersectionObserver对象被创建时，其被配置为监听根中一段给定比例的可见区域。一旦IntersectionObserver被创建，则无法更改其配置，所以一个给定的观察者对象只能用来监听可见区域的特定变化值；然而，你可以在同一个观察者对象中配置监听多个目标元素。

我是这样理解的:   交叉观察者的职责就是观察目标元素与指定元素(默认为窗口视图，即 viewport，可自定义!!)是否发生了交叉行为。通俗来讲，就是观察目标元素是否进入了指定元素的领域，有的话就是交叉，反之。

在这里就不得不为大家展示下我精湛的画技了，图片表达是最容易理解得了；

知道他的作用后,我们还得知道它的食用方式吧!

  首先,因为 IntersectionObserver 是一个构造函数,所以需要将他实例化即可；而且他实例化是接受两个参数，第一个参数为一个回调函数(必填)，第二个参数为一个配置参数对象(选填)。

Callback 回调函数

var intersectionObserver = new IntersectionObserver(function(entries) {
  console.log(entries);
});

此回调函数有一个entries参数,表示处于监听且正发生了交叉的目标集合,是一个数组 entries 数组中的每个元素项都是一个对象,对象包含的属性有:

属性 详情
boundingClientRect 观察元素的空间信息
intersectionRatio 元素可见区域的占比
isIntersecting 是否正在交叉/是否可视
target 当前监听的 Dom 节点 和 event.target (IE 是 event.srcElement)差不多

Option 配置对象

var option = {
  root: document.body,
  rootMargin: "10px 20px 30px 40px"
};

var intersectionObserver = new IntersectionObserver(function(entries) {
  // other code
}, option);

实际应用:

触底加载

以前的触底加载是要不断监听页面和元素的空间信息来判断是否需要发起加载请求更多的数据,光是监听滚动条高频触发就非常耗费性能了,虽然是在可以进行防抖和节流进行优化。不过这些在 IntersectionObserver这个优秀的交叉观察者面前目前都不值一提，因为他是一个异步的 API，光这一点已经完胜。 废话不多说，先敲为敬！

HTML代码

<ul class="data_list"></ul>
<div class="load">触底了，正在加载新数据</div>
1
2

JavaScript代码

var dataView = document.getElementsByClassName("data_list")[0],
  load = document.getElementsByClassName("load")[0];

function loadFn(loadNum) {
  for (var i = 0; i < loadNum; i++) {
    dataView.innerHTML += "<li>这是数据</li>";
  }
  console.log("数据请求完毕");
}

var observe = new IntersectionObserver(function(entries) {
  // 因为监听的只有一个,就直接取出来了
  if (entries[0].isIntersecting) {
    var timer = setTimeout(() => {
      console.log("正在请求数据中...");
      loadFn(10); //加载数据
      clearTimeout(timer); //清除定时器
    }, 1000);
  }
});

observe.observe(load); //监听load

loadFn(10); //初始化加载数据

图片懒加载

其实图片懒加载以往的做法和触底更新差不多,不断计算每个元素的空间位置,判断是否到达了viewport区域,是的话就加载图片资源,那么图片资源去哪里加载呢?? 看下面一段代码:

<img src="loading.png" data-src="srcSource" />
1

创建图片是,先给图片一张加载图片待定,真正的图片资源用一个自定义属性接收,待图片到达可视区域时,就将自定义属性的图片资源赋给图片的src即可.

那么,刚刚看了上图的列表动画,你是不是对图片懒加载基本知道怎么做了呢,其实代码基本差不多. 也就是监听每个元素的空间位置,判断是否显示而已

JavaScript代码

let dataView = document.getElementsByClassName("data_list")[0];

function loadFn(loadNum) {
  for (let i = 0; i < loadNum; i++) {
    dataView.innerHTML +=
      '<li><img src="https://ss0.bdstatic.com/70cFvHSh_Q1YnxGkpoWK1HF6hhy/it/u=1407351685,455271665&fm=26&gp=0.jpg" data-src="https://i0.hdslb.com/bfs/sycp/creative_img/201909/63b427194195dd35e4cb15e46e5bd121.jpg@880w_440h.jpg"></li>';
  }
}

loadFn(100);

let observe = new IntersectionObserver(function(entries) {
  // 因为监听的只有一个,就直接取出来了
  for (let e = 0, el = entries.length; e < el; e++) {
    if (!entries[e].isIntersecting) continue;

    //弄个定时器假装真正网络请求哈哈哈
    let timer = setTimeout(function() {
      //将 img 的 data-src 中的图片资源赋值给 img 的 src
      entries[e].target.src = entries[e].target.dataset.src;
      observe.unobserve(entries[e].target); //注销监听
    }, 1000);
  }
});

let lis = dataView.getElementsByTagName("img");
for (let i = 0; i < lis.length; i++) {
  observe.observe(lis[i]);
}

let td = document.querySelector(".top"),
  nav = document.querySelector(".nav");
let observe = new IntersectionObserver(function(entries) {
  console.log(entries[0].boundingClientRect);
  if (entries[0].boundingClientRect.y < 0) {
    nav.classList.add("fixed");
  } else {
    nav.classList.remove("fixed");
  }
});
