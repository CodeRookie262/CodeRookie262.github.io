    # 什么是迭代器

    # 1. 什么是Iterator(遍历器)

    JavaScript原有的表示"集合"的数据结构,主要是数组(Array)和对象(Object),此外,ES6又新出了 Map 和 Set .JavaScript这样就有了4种数据集合,这样就需要统一的接口机制来处理不同的数据结构.

    遍历器(Iterator)就是这种接口机制,它是一种接口可以为各种不同的数据结构提供统一的访问机制.简言之,就是任何数据结构主要部署了这个 Iterator 接口,就可以遍历该数据结构的所有成员.

    Iterator 的作用有3个:

    1. 为各种数据结构提供一个统一的,简便的访问接口;
    2. 使得数据结构的成员能够按某种次序排列;
    3. 供ES6的遍历命令for...of循环消费
    # Iterator 的遍历过程:
    1. 创建一个指针对象,指向当前数据的起始位置.可以理解成遍历器对象就是一个指针对象;
    2. 第一次调用指针对象的next方法,可以将指针对象指向数据结构的第一个成员;
    3. 第二次调用指针对象的next方法,指针指向数据结构的第二个成员;
    4. 反复调用指针对象的next方法,直到它指向了数据结构的结束位置.

    每一次调用next方法,都会返回当前成员的信息,返回的成员信息是一个包含value和done属性的对象,其中value是当前成员的值,done属性是一个布尔值,表示遍历是否结束 (ps:说到next,很多人第一反应会想到Generator函数,确实,Iterator和Generator有关联)

    下面是模拟next方法返回值的栗子

        var it = myIterator([1,2]);
    it.next();//{value: 1,done: false}
    it.next();//{value: 2,done: false}
    it.next();//{value: undefined,done: true}
    
    
    function myIterator(arr){
        var idx = 0,
            aL = arr.length;
        return {
          next: function(){
            return  (idx) >= aL ? 
            {value: undefined,done: true} : {value: arr[idx++],done: false}
          }  
        };
    }

    # 默认Iterator接口

    Iterator的目的就是为所有数据结构提供一种统一的访问机制,即for...of(下文见).当使用for...of循环遍历某种数据结构时,该循环会自动去寻找Iterator接口.

    只要某种数据结构部署了Iterator接口,我们就称之为这种数据结构是"可遍历的"

    当一种数据结构自带可遍历的功能,那么它就一定拥有Iterator接口,Iterator默认部署在 Symbol.iterator属性中,又或者说,当一个数据结构拥有Symbol.iterator属性,就可以认为是"可遍历的".
    Symbol.iterator属性本身就是一个函数,就是当前数据结构默认的遍历生成函数,执行这个函数时,会返回一个遍历器.至于Symbol.iterator,它是一个表达式,返回Symbol对象的iterator属性,因为它是一个预定义好的,数据类型的Symbol的特殊值,所以需要用object[属性]进行获取
    具备原生 Iterator 接口的数据结构如下

    • Array
    • Map
    • Set
    • String
    • TypedArray
    • 函数的 arguments 对象
    • NodeList 对象

    下面是操作数组的iterator接口的代码

        var arr = [1,2,3];
    var it = arr[Symbol.iterator]();
    it.next();//{value: 1,done: false}
    it.next();//{value: 2,done: false}
    it.next();//{value: 3,done: false}
    it.next();//{value: undefined,done: true}

    对于原生就具备有 Iterator 接口的数据结构,不用自己写遍历器生成器.for...of循环会自动遍历它们,除此之外,其他数据结构(只要是对象)的Iterator接口,都需要自己在Symbol.iterator属性上面部署,这样才会被for...of循环遍历.

    对象(Object)之所以没有默认部署Iterator接口,是因为它是无序的,不知道要先遍历那个属性,哪个属性后遍历,所以需要开发者手动指定
    一个对象如果要具备可被for...of循环调用的Iterator的接口,就必须在其Symbol.itemator的属性上部署遍历器生成方法 下面是对象身上部署Symbol.iterator的代码

        var obj = {a: 1,b: 2};
    
    obj[Symbol.iterator] = function(){
        var idx = 0,
            arr = Object.keys(this),
            ctx = this;
            return {
                next: function(){
                  return  idx >= arr.length ?
                    {value: undefined,done: true}
                     : 
                    {value: ctx[arr[idx++]],done: false}
                }
            }
    }
    
    for(var key of obj){
        console.log(key);
    }
    // 1
    // 2

    # 调用Iterator的场景

    有一些场合会默认调用Iterator接口,除了使用for...of循环,还有几个别的场合也会使用到

    1. 结构赋值 对数组和Set结构进行结构赋值时,会默认调用Symbol.iterator方法
        var set = new Set().add(1).add(2).add(3).add(3);
    var [x,y] = set;
    //x = 1,y = 2
    var [a,...b] = set;
    //a = 1,b = [2,3]

    1. 其他场合
      由于数组的遍历会调用遍历器接口,所以任何接收数组作为参数的场合,其实都调用了遍历器接口,如下一些例子
    • for...of
    • Array.from()
    • Map(),Set()...
    • Promise.all()
    • Promise.race()
    # Iterator接口与Generator函数

    可以利用Grenerator函数实现Symbol.itemator方法

        var arr = {
        [Symbol.iterator]: function* (){
            yeild 1;
            yeild 2;
        }
    }
    [...arr];
    //[1,2]
    var obj = {
        [Symbol.iterator]: function* (){
            yeild "a";
            yeild "b";
        }
    }
    for(var key of obj){
        console.log(key);
    }
    // "a"
    // "b"
