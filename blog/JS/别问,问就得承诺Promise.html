# 别问,问就得承诺(Promise)

承诺的英语是Promise,巧了,我们今天的主题也和Promise有关(滑稽~)!

在进入正式主题之前,我们先给大伙们看一段代码案例,想必各位都知道JS是单线程.这样的话,当某一事件正在进行中,会阻碍后面代码的运行,但是现实却不会,这是为什么呢?原因很简单,因为事件分为了同步事件和异步事件,这就是但我们在发起网络请求的时候,不会影响其他代码的运行的原因.也就是说,网络请求属于一种异步事件,说到底,我们不知道他什么时候会成功,只能在他返回结果的那一刻才能保证他是执行完毕的.

假如我们有多个带有关联性的异步请求时,在没有Promise前我们可能会这样做

    //发起请求函数
    function ajax(option){...}
    
    ajax({
        url: "./log.php",
        data: {...},
        success: function(user){
            //已保证用户消息的获取后发起二次请求
            ajax({
                url: "./artic.php",
                data: user,
                success: function(mes){
                    //对获取到的 msg 就行渲染
                    //渲染代码块 Code ...
                }
            })
        }
    })

什么是 Promise

Promise 是异步编程的一种解决方案，比传统的解决方案——回调函数和事件——更合理和更强大。

所以Promise，简单说就是一个容器，里面保存着某个未来才会结束的事件（通常是一个异步操作）的结果。

特点

  • 对象的状态不受外界影响。
  • 一旦状态改变，就不会再变，任何时候都可以得到这个结果。

状态

Promise对象代表一个异步操作，它有着三种状态: Pending(进行中)，Fulfilled(成功)和Rejected(失败)。

PS: 只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态。

缺点

  1. 无法取消Promise。一旦新建它就会立刻执行，无法中途取消。
  2. 如果不设置回调函数，Promise内部会抛出错误，且不会反应到外部
  3. 当Promise的状态处于Pending时，是无法得知当前的进展阶段的

注意，为了行文方便，本章后面的 resolved 统一只指 fulfilled 状态，不包含 rejected 状态。

Promise实例生成以后，可以用then方法分别指定resolved状态和rejected状态的回调函数。

链式操作的用法

所以，从表面上看，Promise只是能够简化层层回调的写法，而实质上，Promise的精髓是“状态”，用维护状态、传递状态的方式来使得回调函数能够及时调用，它比传递callback函数要简单、灵活的多。所以使用Promise的正确场景是这样的：

function runAsync1(){
    var p = new Promise(function(resolve, reject){
        //做一些异步操作
        setTimeout(function(){
            console.log('异步任务1执行完成');
            resolve('随便什么数据1');
        }, 1000);
    });
    return p;            
}
function runAsync2(){
    var p = new Promise(function(resolve, reject){
        //做一些异步操作
        setTimeout(function(){
            console.log('异步任务2执行完成');
            resolve('随便什么数据2');
        }, 2000);
    });
    return p;            
}
function runAsync3(){
    var p = new Promise(function(resolve, reject){
        //做一些异步操作
        setTimeout(function(){
            console.log('异步任务3执行完成');
            resolve('随便什么数据3');
        }, 2000);
    });
    return p;            
}

runAsync1()
    .then(function(data){
        console.log(data);
        return runAsync2();
    })
    .then(function(data){
        console.log(data);
        return runAsync3();
    })
    .then(function(data){
        console.log(data);
    });
//结果: 
//异步任务1执行完成
//随便什么数据1
//异步任务2执行完成
//随便什么数据2
//异步任务3执行完成
//随便什么数据3

当然,实例化的Promise的then内部有两个回调函数,第一个回调函数只有在成功状态下的时候才会执行,第二个回调函数只有在失败状态下才会执行,用于监听捕获错误信息.

但是,Promise实例类监听捕获错误信息有专门的方法,那就是Promise.prototype.catch

Promise.prototype.catch

Promise.prototype.catch方法是.then(null, rejection)的别名，用于指定发生错误时的回调函数。 看下下面的栗子:

var p1 = new Promise(function(resolve, reject) {
  resolve('Success');
});

p1.then(function(value) {
  console.log(value); // "Success!"
  throw 'oh, no!';
}).catch(function(e) {
  console.log(e); // "oh, no!"
})

//结果
//Success
//oh, no!

同时,Promise也有很多静态方法,分别是resolve,reject,all和race

Promise.all()

Promise.all方法用于将多个 Promise 实例，包装成一个新的 Promise 实例。该方法接收一个参数，参数为一个数组且数组的每个元素都为Promise实例类，该方法支持then操作。当传入的数组中所有Promise的状态都为fulfilled(成功状态)时，才会执行then的成功回调，如果传入的Promise中有一个状态为rejected的话，则判定为rejected状态，则会执行then的失败回调函数。

但如果你有多个操作都需要成功的情况下才可以执行某个事件的情况下，最适合用Promise.all()

var p1 = new Promise(function(resolve,reject){
    $.ajax({
        url: ...,
        data: ...,
        success: resolve,
        error: reject
    })
});

var p2 = new Promise(function(resolve,reject){
    $.ajax({
        url: ...,
        data: ...,
        success: resolve,
        error: reject
    })
});

Promise
    .all([p1,p2])
    .then(function(res){
        //res是一个结果集合
        console.log(res)
    },function(err){
        console.log("失败了")
    })

Promise.race() Promise.race方法同样是将多个 Promise 实例，包装成一个新的 Promise 实例。

var p = Promise.race([p1, p2, p3]);
1

上面代码中，只要p1、p2和p3之中有一个实例率先改变状态，p的状态就跟着改变。那个率先改变的Promise 实例的返回值，就传递给p的回调函数。

Promise.resolve()

有时需要将现有对象转为 Promise 对象，Promise.resolve方法就起到这个作用。

var jsPromise = Promise.resolve('HelloWorld');
1

上面代码将'HelloWorld'转为一个 Promise 对象。Promise.resolve等价于下面的写法。

Promise.resolve('HelloWorld')
// 等价于
new Promise(resolve => resolve('HelloWorld'))

Promise.resolve方法的参数分成四种情况。

# 参数是一个 Promise 实例

如果参数是 Promise 实例，那么Promise.resolve将不做任何修改、原封不动地返回这个实例。

# 参数是一个thenable对象

thenable对象指的是具有then方法的对象，比如下面这个对象。

var thenable = {
  then: function(resolve, reject) {
    resolve(2333);
  }
};
1

5

Promise.resolve方法会将这个对象转为 Promise 对象，然后就立即执行thenable对象的then方法。

var thenable = {
  then: function(resolve, reject) {
    resolve(42);
  }
};

var p1 = Promise.resolve(thenable);
p1.then(function(value) {
  console.log(value);  // 42
});

# 参数不是具有then方法的对象，或根本就不是对象

如果参数是一个原始值，或者是一个不具有then方法的对象，则Promise.resolve方法返回一个新的 Promise 对象，状态为resolved。

var p = Promise.resolve('Hello');

p.then(function (s){
  console.log(s)
});
// Hello

# 不带有任何参数

Promise.resolve方法允许调用时不带参数，直接返回一resolved状态的 Promise 对象。 所以，如果希望得到一个Promise 对象，比较方便的方法就是直接调用Promise.resolve方法。

var p = Promise.resolve();

p.then(function (res) {
  console.log("success")
});

//success

Promise.reject(reason)方法也会返回一个新的 Promise 实例，该实例的状态为rejected。

看了这么多,我们先来利用Promise封装一个sleep函数吧

function sleep(time,param){
    time = time ? (time < 0 ? 0 : time) : 0;
    return new Promise(function(resolve){
        setTimeout(resolve,time,param)
    })
}

function say(content){
    console.log(content)
}
//延迟 3s 调用 say 函数
sleep(3000,"HelloWorld").then(say)
